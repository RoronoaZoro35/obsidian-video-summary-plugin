# 缓存功能测试

这是一个用于测试视频总结插件缓存功能的示例文件。

## 测试步骤

### 1. 首次处理（会调用API）
- 使用命令面板调用 "📺 快速总结当前笔记"
- 选择 "完整总结" 模式
- 选择语言（中文/英文/日文）
- 等待处理完成

### 2. 重复处理（应该使用缓存）
- 再次使用命令面板调用 "📺 快速总结当前笔记"
- 选择相同的模式和语言
- 应该立即返回结果，无需等待API调用

### 3. 不同模式处理（会调用API）
- 使用命令面板调用 "📝 快速提取文稿"
- 选择相同的语言
- 应该调用API，因为模式不同

### 4. 不同语言处理（会调用API）
- 使用命令面板调用 "📺 快速总结当前笔记"
- 选择不同的语言
- 应该调用API，因为语言不同

## 预期结果

- 相同URL + 相同模式 + 相同语言 = 使用缓存
- 不同URL = 调用API
- 不同模式 = 调用API  
- 不同语言 = 调用API

## 缓存设置

可以在插件设置中：
- 启用/禁用缓存
- 设置缓存过期时间
- 查看缓存统计
- 清理过期缓存
- 清空所有缓存

## 费用节省效果

- 重复处理相同视频：100%节省API费用
- 批量处理包含重复视频：大幅节省费用
- 多语言处理：部分节省费用
- 长期使用：累积节省大量费用
