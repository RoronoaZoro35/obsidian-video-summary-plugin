# 智能缓存系统详细说明

## 概述

智能缓存系统是视频总结插件的核心功能之一，旨在通过缓存已处理的视频结果来大幅节省API调用费用，同时提升用户体验。

## 工作原理

### 缓存键生成
缓存系统使用以下三个维度的组合作为唯一键：
- **URL**: 视频链接地址
- **处理模式**: `summary`（完整总结）或 `transcript-only`（仅提取文稿）
- **语言**: `zh`（中文）、`en`（英文）、`ja`（日文）

```
缓存键 = `${url}|${mode}|${language}`
```

### 缓存流程
1. **处理前检查**: 检查是否存在匹配的缓存
2. **缓存命中**: 直接返回缓存结果，无需调用API
3. **缓存未命中**: 调用API处理，将结果存入缓存
4. **过期清理**: 自动清理过期的缓存项

## 配置选项

### 启用/禁用缓存
- **默认状态**: 启用
- **作用**: 控制是否使用缓存功能
- **建议**: 建议保持启用状态以节省费用

### 缓存过期时间
- **默认值**: 30天
- **范围**: 1-365天
- **作用**: 控制缓存结果的有效期
- **建议**: 根据视频内容更新频率调整

### 缓存容量
- **默认值**: 1000项
- **作用**: 限制内存中缓存的最大数量
- **策略**: 达到上限时自动删除最旧的缓存项

## 使用场景

### 1. 重复处理
**场景**: 用户多次处理同一个视频
**效果**: 100%节省API费用
**示例**: 重新生成总结、切换语言等

### 2. 批量处理
**场景**: 批量处理包含重复视频的文件
**效果**: 大幅节省API费用
**示例**: 播放列表处理、文件夹批量处理

### 3. 多语言处理
**场景**: 同一视频生成不同语言版本
**效果**: 部分节省API费用
**示例**: 中文总结 + 英文总结

### 4. 模式切换
**场景**: 同一视频的不同处理模式
**效果**: 部分节省API费用
**示例**: 完整总结 + 仅提取文稿

## 缓存管理

### 查看统计
显示当前缓存的使用情况：
- 已缓存项数
- 最大容量
- 过期项数

### 清理过期
自动删除已过期的缓存项：
- 释放内存空间
- 保持缓存时效性
- 建议定期执行

### 清空缓存
删除所有缓存项：
- 谨慎操作
- 适用于重置场景
- 会清空所有缓存数据

### 响应速度
- **缓存命中**: 毫秒级响应

- **总体提升**: 显著提升重复处理速度

## 费用节省

### 计算示例
假设API调用费用为 $0.01/次：

**无缓存情况**:
- 处理10个相同视频: $0.10
- 重复处理5次: $0.50
- 总计: $0.60

**有缓存情况**:
- 首次处理10个视频: $0.10
- 重复处理5次: $0.00（使用缓存）
- 总计: $0.10
- **节省**: $0.50 (83.3%)

## 故障排除

### 缓存不生效
1. 检查是否启用了缓存
2. 确认URL格式一致
3. 验证模式和语言设置
4. 查看控制台日志

### 缓存占用过高
1. 减少缓存容量
2. 缩短过期时间
3. 定期清理缓存
4. 监控内存使用

### 缓存数据丢失
1. 检查本地存储权限
2. 验证浏览器设置
3. 查看错误日志
4. 重新初始化缓存

## 技术细节

### 存储方式
- **内存存储**: 使用Map数据结构
- **持久化**: 保存到localStorage
- **序列化**: JSON格式存储

### 过期机制
- **时间戳**: 记录创建和过期时间
- **自动清理**: 访问时检查过期状态
- **批量清理**: 支持手动清理过期项

### 并发安全
- **读写分离**: 缓存操作不影响API调用
- **原子操作**: 缓存更新是原子性的
- **错误处理**: 缓存失败不影响主要功能
