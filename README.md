# 视频总结插件

一个强大的Obsidian插件，用于自动生成视频内容的总结和文稿。

## ✨ 新功能

### 🚀 实时批量处理
- **实时结果显示**：批量处理时，每完成一个文件就立即显示结果，无需等待所有文件完成
- **错误隔离**：单个文件处理失败不会影响其他文件的处理结果
- **实时进度**：显示当前处理状态、成功数量、失败数量等详细信息
- **即时反馈**：每个文件处理完成后立即显示成功/失败通知

### 💾 智能缓存系统
- **API费用节省**：相同视频链接的处理结果会被缓存，避免重复调用API
- **智能过期管理**：缓存结果自动过期，确保数据时效性
- **多维度缓存**：基于URL、处理模式、语言的组合进行缓存
- **缓存管理**：支持查看统计、清理过期、清空缓存等操作
- **可配置性**：可设置缓存过期时间，支持启用/禁用缓存

## 📁 缓存存储位置

### 当前版本 (v2.1.2+)
缓存文件存储在插件的专用数据目录中：
```
.obsidian/plugins/video-summary-plugin/data/cache/
├── index.json          # 缓存索引文件
└── items/              # 缓存项文件目录
    ├── abc123.json     # 缓存项文件
    └── ...
```

### 之前版本
缓存文件存储在库根目录的 `cache/` 文件夹中。

**注意**：插件会自动迁移旧缓存到新位置，无需手动操作。

详细说明请查看：[缓存存储位置说明](docs/CACHE_STORAGE_LOCATION.md)

## 架构原理

**⚠️ 注意：本插件必须联合 [n8n](https://n8n.io/) 一起使用！**

这不仅仅是一个单纯的前端插件。插件主要负责在 Obsidian 内提供友好的 UI、触发请求和解析最终排版结果，而**所有繁重的视频下载、音频提取、语音转文字以及大模型深度总结排版**都是在 n8n 服务端完成的。

工作流程如下：
1. **Obsidian** 发送视频链接等元数据到 n8n Webhook。
2. **n8n** 内部通过 `yt-dlp` 下载视频/音频，并使用 AI 服务提取文稿。
3. **n8n** 将文稿发送给大模型 (Gemini / OpenAI 等) 按照预设的系统提示词（已配置在提供的 `Obsidian Video Summary.json` 文件中）生成精美的 Obsidian 笔记格式 (Markdown)。
4. 返回给 **Obsidian** 自动插入或替换当前笔记。

我们已将完整、经过无数次优化的 n8n Workflow 文件 (`Obsidian Video Summary.json`) 包含在本项目中。你只需在 n8n 中导入它并填入你自己的 API Key 即可。

## 使用方法

### 单个视频处理
1. 在笔记中添加视频链接
2. 使用命令面板或快捷键调用视频总结
3. 选择处理模式和语言
4. 等待处理完成

### 批量处理
1. 打开批量处理模态框
2. 选择要处理的文件或播放列表
3. 设置处理参数和并发数
4. 开始处理，实时查看进度和结果

## 配置选项

- **API设置**：配置n8n webhook地址
- **处理选项**：设置默认语言、处理模式、超时时间
- **批量处理**：配置并发数量
- **文件管理**：自动重命名、冲突处理策略
- **缓存设置**：启用/禁用缓存、设置过期时间、管理缓存
- **高级选项**：调试模式、重试机制

## 安装

1. 下载插件文件
2. 放入Obsidian插件目录
3. 在Obsidian中启用插件
4. 配置API设置
